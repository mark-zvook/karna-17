# KARNA-17 — RTC configuration notes
#
# The CM5 has a built-in RTC in the RP1 south bridge (driver: rpi-rtc).
# It registers as /dev/rtc0 automatically — NO dtoverlay is needed.
#
# The baseboard has a CR1220 coin cell holder wired to the CM5 VRTC pin.
# Inserting the battery is all that is needed for backup timekeeping.
#
# I2C bus 1 devices (for reference):
#   0x50  — EEPROM (CM5 module)
#   0x64  — unknown (baseboard peripheral)
#   0x68  — MPU-6500 IMU
#   (no external RTC chip on bus — rpi-rtc is not I2C-accessible from userspace)
#
# What setup_rtc.sh does:
#   1. Disables fake-hwclock (conflicts with real RTC)
#   2. Enables systemd-timesyncd for NTP
#   3. Installs karna-rtc-sync.path/.service (hwclock --systohc after NTP sync)
